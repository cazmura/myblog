仮想環境を立ち上げて、pip でパッケージなどをインストールするまでの流れをまとめる。

プロジェクトのディレクトリ上で以下のコマンドを叩く。

```
python3 -m venv myenv
source myenv/bin/activate
pip install パッケージ名
```

1行ずつ解説していく

- python3 -m venv myenv
	- このコマンドは、Pythonの仮想環境を作成します。`-m venv`は、Pythonの標準モジュールである`venv`を実行するという意味で、`myenv`は作成する仮想環境のディレクトリ名です。仮想環境を使うと、プロジェクトごとに独立したPython環境を作れるため、異なるプロジェクト間でパッケージのバージョンが競合するのを防げます。
	  
- source myenv/bin/activate
	- このコマンドは、作成した仮想環境を有効化します。`source`コマンドでアクティベーションスクリプトを実行することで、シェルのPython環境が仮想環境に切り替わります。有効化すると、プロンプトに`(myenv)`のような表示が追加され、この後にインストールするパッケージは仮想環境内にのみインストールされます。
	  
- pip install パッケージ名
	- このコマンドは、有効化された仮想環境内に指定したPythonパッケージをインストールします。`pip`はPythonのパッケージ管理ツールで、PyPI（Python Package Index）から必要なライブラリやツールをダウンロードしてインストールします。「パッケージ名」の部分には、実際にインストールしたいパッケージの名前（例: `requests`、`numpy`など）を指定します。

### 補足
- プロジェクトごとに、同じ仮想環境のディレクトリ名を使っても問題ない
	- 仮想環境を切り替えたい時は、プロジェクトのディレクトリ上で`source...` のコマンドを叩けば切り替わる
- 仮想環境の終了の仕方は`deactivate`を叩く
	- 仮想環境を終了（`deactivate`）すると、その仮想環境内にインストールしたパッケージは使えなくなる
		- ただし、パッケージは削除されていない。仮想環境のディレクトリ（`venv/` など）の中にちゃんと保存されている
			- 再度 `source venv/bin/activate` で仮想環境を有効化すれば、また同じパッケージが使えるようになる
