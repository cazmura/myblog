よく、pip xxx xxxx とか、npm xxx xxx とか、こういったコマンドを叩く場面に遭遇する。

これらは何をしているのかというと、パッケージをインストールしている。

パッケージとは、他人が書いたコード集のこと。  
パッケージをインストールすることで、機能を0から実装しなくても良くなる。

pip など経由でインストールする利点は以下
- 統一された方法でインストールできる
- 依存関係のあるパッケージも自動でインストールしてくれる
- バージョン管理が楽（らしい）
- 信頼性と安全性がある
- 環境の再現が楽

### インストール先ってどこ？

pip と npm でパッケージのインストール先が異なる。

pip はどのディレクトリからインストールのコマンドを叩いても、システム全体にインストールされる。つまり、影響範囲がシステム全体に渡る。

一方、npm はインストールのコマンドを叩いたディレクトリの node_modules にインストールされる。影響範囲はディレクトリの中のみ。

そのため、pip を特定のプロジェクトのみで有効にしたい場合、仮想環境というものを使い、そこにパッケージをインストールする必要がある。
